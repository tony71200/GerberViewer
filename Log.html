GerberViewer Implementation Log

All function and variable changes made for the Online Gerber Viewer-style UI/spec implementation are listed here.

FileFunction / variableChangeReason
GerberViewer/MainForm.csPreviewDpiAdded fixed preview DPI constant at 300.Keeps preview independent from Export DPI and avoids high-resolution preview allocations.
GerberViewer/MainForm.csLargePrimitiveWarningThresholdAdded 50,000 primitive threshold.Surfaces large-scene performance status for high-resolution Gerber files.
GerberViewer/MainForm.csLoadFilesTracks primitive counts and displays Parsing/large-scene status in English.Improves logic/performance visibility and status-state clarity.
GerberViewer/MainForm.csBuildOptions(bool forPreview)Preview rendering now uses PreviewDpi; exports use the selected Export DPI.Prevents Export DPI from changing preview scale/sharpness and reduces memory for large boards.
GerberViewer/MainForm.csRenderPreviewAsyncStatus text changed to “Generating preview” and “Ready (preview transform, Export DPI independent)”.Aligns with spec state language.
GerberViewer/GerberCanvas.cs_majorGridPen, _minorGridPenAdded reusable grid pens.Creates dark CAD-like grid similar to onlinegerberviewer.com.
GerberViewer/GerberCanvas.csOnPaintDraws dark grid before image and updates empty-state text.Improves visual similarity to reference UI.
GerberViewer/GerberCanvas.csDrawGrid(Graphics g)Added helper for minor/major grid rendering.Keeps grid drawing isolated and reusable.
GerberViewer/GerberCanvas.csDispose(bool disposing)Disposes grid pens in addition to preview bitmap.Prevents GDI resource leaks.
GerberViewer/MainForm.Designer.csInitializeComponent UI labels/styleRenamed toolbar labels, constrained Export DPI choices, applied dark toolbar/sidebar and light status bar.Matches spec-required controls and visual style of onlinegerberviewer.com.
GerberViewer/GerberCanvas.csGerberCanvas : EWindowControl.EWindowControl, SetImage, FitToView, ImageCursorMovedChanged GerberCanvas into an adapter over EWindowControl while preserving the existing ReadGerberControl API.Matches the referenced branch by moving Gerber preview display/zoom/pan to EWindowControl without changing callers that render to Bitmap.
EWindowControl/EWindowControl.csHasSourceImage, CurrentZoom, ImagePointMoved, ClearImage, SetSourceBitmapAdded public adapter members so GerberCanvas can pass Bitmap images into the HALCON-backed control and report image coordinates.Provides the Bitmap-to-HALCON bridge in EWindowControl and supports the GerberCanvas adapter from the referenced branch.
GerberViewer/Views/ReadGerberControl.csRenderPreviewAsyncRestored preview rendering to call BuildOptions(true) and canvas.SetImage(...).Keeps the existing Bitmap render path and lets GerberCanvas/EWindowControl handle display.
GerberViewer/GerberViewer.csprojhalcondotnetxl referenceAdded explicit HALCON .NET reference resolution through HalconDotNetPath, preferring $(HALCONROOT)\bin\dotnet35\halcondotnetxl.dll and falling back to the bundled DLL, with SpecificVersion disabled and Private enabled.Fixes CS0012 in GerberViewer designer-generated controls such as CreateGerberSampleControl that instantiate EWindowControl and therefore require the HALCON assembly at compile time.
EWindowControl/EWindowControl.csprojHalconDotNetPath, halcondotnetxl referenceAligned the EWindowControl project with the same HALCON reference resolution used by GerberViewer.Prevents design-time/project-reference builds from resolving EWindowControl against a different HALCON assembly identity.
GerberStitching.Core/Configuration/GerberSampleConfig.csGerberSampleConfigMarked persisted config properties with data-contract metadata while excluding runtime-only PadColor.Keeps fixed-path JSON serialization focused on stable configuration fields.
GerberStitching.Core/Configuration/AppPaths.cs, SampleConfigStore.csAppPaths.SampleConfigPath, SampleConfigStore.LoadOrCreateDefault, SaveAdded fixed-path config load/create/save with directory creation, temp-file readback, and invalid JSON backup.Implements Tab 2 fixed config flow and removes config persistence from the UI.
GerberStitching.Core/Imaging/SampleGridGeometry.csSampleGeometryCalculator.Calculate, SampleGridLayout, SampleTileStateAdded reusable deterministic grid layout and per-tile state model.Lets Tab 2 render pending/completed tile overlays immediately after image load or config changes.
GerberViewer/Views/CreateGerberSampleControl.Designer.csbtnSaveConfig, commandLayoutAdded Save Config next to Load Config and moved the command strip to a non-overlapping table layout.Matches the required fixed Tab 2 control order and preserves relative positions on resize.
GerberViewer/Views/CreateGerberSampleControl.csLoadConfigOnStartup, btnOpenSample_Click, btnLoadSampleConfig_Click, btnSaveConfig_Click, RenderGridOverlayStartup/load/save now use SampleConfigStore; selected rasters are decoded and displayed; grid labels are drawn red and completed progress turns tiles green.Fixes Tab 2 path-only image loading and gives every tile explicit UI state instead of relying only on progress total.
GerberStitching.Core/Imaging/SampleTileGenerator.csGenerateReplaced empty marker-write catch with contextual exception reporting.Prevents swallowed exceptions during failed/cancelled sample generation.
GerberViewer/Views/CreateGerberSampleControl.csCommitWorkflowContextMapped Tab 2 GerberSampleConfig.SourceRasterPath into the existing workflow SampleGerberConfig instead of assigning incompatible config types.Fixes CS0029 while preserving the shared WorkflowContext contract used by other tabs.
EWindowControl/EWindowControl.csSetSourceImage, RenderImageOverlayAdded HALCON-image display ownership and image-coordinate overlay rendering APIs for large TIFF/BigTIFF samples.Avoids loading large sample rasters through System.Drawing.Bitmap for Tab 2 preview and fixes TIFF decode failures from GDI+.
GerberViewer/Views/CreateGerberSampleControl.csbtnOpenSample_Click, ReplaceSampleImage, RenderGridOverlayChanged Tab 2 sample loading from ImageRead.ReadBitmap to HOperatorSet.ReadImage, stores HObject plus HALCON dimensions, and renders overlays through EWindowControl.RenderImageOverlay.Supports large TIFF/BigTIFF input without the Bitmap..ctor parameter error and avoids rasterizing the overlay into a huge bitmap.
EWindowControl/EWindowControl.csRenderImageOverlayChanged sample overlay rectangles to HALCON margin draw mode and rendered order labels in yellow boxed text.Fixes filled red rectangles hiding the image and makes tile order numbers visible.
