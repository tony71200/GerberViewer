GerberViewer Implementation Log

All function and variable changes made for the Online Gerber Viewer-style UI/spec implementation are listed here.

FileFunction / variableChangeReason
GerberViewer/MainForm.csPreviewDpiAdded fixed preview DPI constant at 300.Keeps preview independent from Export DPI and avoids high-resolution preview allocations.
GerberViewer/MainForm.csLargePrimitiveWarningThresholdAdded 50,000 primitive threshold.Surfaces large-scene performance status for high-resolution Gerber files.
GerberViewer/MainForm.csLoadFilesTracks primitive counts and displays Parsing/large-scene status in English.Improves logic/performance visibility and status-state clarity.
GerberViewer/MainForm.csBuildOptions(bool forPreview)Preview rendering now uses PreviewDpi; exports use the selected Export DPI.Prevents Export DPI from changing preview scale/sharpness and reduces memory for large boards.
GerberViewer/MainForm.csRenderPreviewAsyncStatus text changed to “Generating preview” and “Ready (preview transform, Export DPI independent)”.Aligns with spec state language.
GerberViewer/GerberCanvas.cs_majorGridPen, _minorGridPenAdded reusable grid pens.Creates dark CAD-like grid similar to onlinegerberviewer.com.
GerberViewer/GerberCanvas.csOnPaintDraws dark grid before image and updates empty-state text.Improves visual similarity to reference UI.
GerberViewer/GerberCanvas.csDrawGrid(Graphics g)Added helper for minor/major grid rendering.Keeps grid drawing isolated and reusable.
GerberViewer/GerberCanvas.csDispose(bool disposing)Disposes grid pens in addition to preview bitmap.Prevents GDI resource leaks.
GerberViewer/MainForm.Designer.csInitializeComponent UI labels/styleRenamed toolbar labels, constrained Export DPI choices, applied dark toolbar/sidebar and light status bar.Matches spec-required controls and visual style of onlinegerberviewer.com.
GerberViewer/GerberCanvas.csGerberCanvas : EWindowControl.EWindowControl, SetImage, FitToView, ImageCursorMovedChanged GerberCanvas into an adapter over EWindowControl while preserving the existing ReadGerberControl API.Matches the referenced branch by moving Gerber preview display/zoom/pan to EWindowControl without changing callers that render to Bitmap.
EWindowControl/EWindowControl.csHasSourceImage, CurrentZoom, ImagePointMoved, ClearImage, SetSourceBitmapAdded public adapter members so GerberCanvas can pass Bitmap images into the HALCON-backed control and report image coordinates.Provides the Bitmap-to-HALCON bridge in EWindowControl and supports the GerberCanvas adapter from the referenced branch.
GerberViewer/Views/ReadGerberControl.csRenderPreviewAsyncRestored preview rendering to call BuildOptions(true) and canvas.SetImage(...).Keeps the existing Bitmap render path and lets GerberCanvas/EWindowControl handle display.
GerberViewer/GerberViewer.csprojhalcondotnetxl referenceAdded an explicit HALCON .NET reference pointing to ..\EWindowControl\dll\halcondotnetxl.dll with SpecificVersion disabled.Fixes CS0012 in GerberViewer designer-generated controls that instantiate EWindowControl and therefore require the HALCON assembly at compile time.
