GerberStitching.Core/Alignment

  • ISampleAligner: interface align sample tile, trả SampleAlignmentResult; direction transform chuẩn là Captured → Sample.
  • SampleAlignmentContext: input alignment gồm SampleTileId, SampleNccModelPath, SampleImage, CapturedImage, ExpectedCapturedToSampleTransform, InitialCapturedToSampleTransform và Options; ảnh gốc không bị overwrite.
  • SampleAlignmentResult: result gồm Success, CapturedToSampleTransform 3x3 CV_64F, NccScore, EccCorrelation, rotation/translation/scale, OverlapRatio, Method, PreprocessingVariant, RejectionReason, Warning và optional DiagnosticImages.
  • SampleAlignmentOptions: thresholds/config gồm NccMinScore, EccMinCorrelation, MinOverlapRatio, MaxTranslationPixels, MaxAbsRotationDeg, MinScale, MaxScale, AllowNccOnlyAcceptance, AllowEccFromExpectedWhenNccFails, PyramidLevels, EccIterations, EccEpsilon và Preprocessing.
  • PreprocessingOptions: modality-aware pipeline gồm grayscale conversion, ContrastNormalization, polarity sample/captured (PolarityMode), threshold fixed/Otsu/adaptive (ThresholdMode), Sobel/Canny/HALCON-equivalent edges (EdgePreparationMode), Gerber content mask, normalized size và diagnostic image flag.
  • ModalityAwarePreprocessor.Preprocess: tạo bản processed mới cho sample/captured, không mutate bitmap input; xuất PreprocessedAlignmentImages disposable.
  • HalconNccSampleAligner: NCC coarse aligner với hook/cache per SampleTileId tương ứng create_ncc_model/find_ncc_model, convert row/column/angle sang homography Captured → Sample, dispose cache deterministic và reject khi score nhỏ hơn NccMinScore.
  • PyramidEccSampleAligner: ECC refinement path chuẩn bị ảnh single-channel float, dùng expected/NCC init, validate correlation/translation/rotation/scale/overlap/finiteness và giữ transform 3x3 CV_64F.
  • NccThenPyramidEccSampleAligner: hybrid mặc định chạy NCC rồi ECC, accept khi cả hai pass, accept NCC-only khi geometry/threshold pass với warning NccOnlyAccepted, và optionally chạy ECC từ expected-grid init nếu NCC fail.
  • NeighborMatchAcceptance.IsAccepted: helper migration cho neighbor matcher, success chỉ khi pairResult?.Eval?.IsMatch == true; không accept chỉ vì Eval != null.
  • SampleAlignmentMethod, PolarityMode, ThresholdMode, EdgePreparationMode, ContrastNormalizationMode: enums cho method/preprocessing variant.

Tab 2 overlay and overlap variables

  • SampleGridLayout.TileWidth and TileHeight: desired per-tile dimensions; when GerberSampleConfig.ProcessedWidth/ProcessedHeight are greater than zero they are treated as tile size, otherwise fallback geometry is derived from the actual processed image size.
  • overlapX and overlapY: overlap measured between tiles; percent overlap is calculated from TileWidth/TileHeight, not from the full processed image.
  • SampleGridLayout.StepX and StepY: distance between tile starts, calculated as TileWidth - overlapX and TileHeight - overlapY; for 4096 tile and 60 px overlap the step is 4036.
  • SampleGridLayout.Warnings: non-blocking warnings such as unused processed width/height or tile grid overflow; rectangles are clamped to processed image bounds before preview/crop/manifest output.
  • AddRectangleOutline: creates HALCON rectangle-region outlines so the overlay is outline-only/margin-style.
  • AddFilledRect: creates the HALCON rectangle regions used by tile borders and label/background helper shapes.
  • AddLabelTextRegions: creates normal bold-font OrderIndex label regions with a dark inverse/contrast border and light fill instead of seven-segment digital digits.
  • DrawTextMask and AddMaskRegion: render small owned GDI+ text masks and convert active runs into HALCON regions, avoiding full-size image bitmap conversion.

Tab 2 zoom-aware overlay variables

  • CurrentZoom: inherited viewer zoom value used by RenderImageOverlay to scale overlay geometry.
  • lineWidth: image-coordinate rectangle and label-outline thickness calculated from CurrentZoom but capped to avoid generating oversized overlay regions at low zoom.
  • labelSize: image-coordinate normal-font order-number size calculated from CurrentZoom but capped before GDI+ mask creation to prevent preview hangs.

Tab 2 NCC model output variables

  • NccModelPath: per-tile manifest field pointing to tiles/<basename>.ncm; required for manifest v3 validation when files are required.
  • SampleTileGenerator.WriteNccModel: creates a HALCON NCC model from the cropped tile using MatcherOptions angle/level/metric defaults, writes it via HOperatorSet.WriteNccModel, and clears the model handle in finally.
  • VerifyNccModelReadable: readback check using HOperatorSet.ReadNccModel followed by HOperatorSet.ClearNccModel to validate the published model can be consumed by Tab 3.

Tab 3 NCC model consumption variables

  • MatchRequest.ReferenceNccModelPath: optional path to a prebuilt HALCON NCC model for the reference/sample tile; if absent or unreadable, the matcher keeps the compatibility fallback and creates a model from ReferenceImage.
  • NCC_HalconMatcher.GetOrCreateModel: loads ReferenceNccModelPath with HOperatorSet.ReadNccModel when available, otherwise calls CreateNccModel from the reference image.
  • NCC_HalconMatcher.BuildCacheKey: cache key includes sample tile identity, preprocessing variant, NCC options, reference image metadata, normalized model path, and model file last-write/size stamp to avoid stale model reuse.

Tab 3 align/stitch workflow variables

  • AlignStitchConfig.InputManifestPath, CapturedFolderPath, and OutputPath: selected manifest, captured-image folder, and user output root; output root is never recursively deleted.
  • AlignStitchConfig.NccMinScore, EccMinCorrelation, MaxTranslationPixels, MaxAbsRotationDeg, MinScale, MaxScale, and MinOverlapRatio: direct camera-to-sample acceptance thresholds edited in alignConfigGrid.
  • AlignStitchConfig.EnableNeighborRecovery, EnableAnchorInterpolation, AllowExpectedGridFallback, and RequireManualConfirmationForExpectedGrid: recovery controls; production output remains blocked when these paths are unavailable or unverified.
  • PathCanvasNode.NodeId: stable UI node identity equal to CapturedImageInfo.OrderIndex and SampleTileInfo.OrderIndex for loaded manifest/captured data.
  • PathCanvasSnapshot: immutable paint snapshot used by the WinForms order/status canvas.
  • TileWorkflowState.OrderIndex and ProcessingTileReport.OrderIndex: structured diagnostics now carry canonical tile identity in addition to row/column.
  • NccModelHandle: explicit NCC model-cache handle metadata replacing fake new object() success placeholders.
